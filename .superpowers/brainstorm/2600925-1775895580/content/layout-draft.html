版式说明

  • 主流程：从上到下，业务接入 → 控制编排 → 数据平面
  • 观测域：放在右侧独立区域，避免和主控制链路交叉过多
  • 箭头组织：
    • 蓝色实线：控制下发
    • 橙色虚线：状态反馈
    • 绿色粗线：业务流量（数据平面内部）
  • 分组框：每一层用不同颜色的容器框包裹，便于区分职责
  • 接口标注：在箭头旁边标注协议名称（REST / gNMI / PCEP 等）

draw.io 实施建议

  • 使用 Container 形状作为每一层的外框
  • 模块用 Rectangle，圆角 4-6px
  • 箭头用 Connector，设置不同样式和颜色
  • 协议标注用 Label 直接贴在箭头上
  • 建议画布尺寸：A3 横向（420mm × 297mm）或更大
  • 导出时选择 PNG 高分辨率 或 SVG 格式
